# KQL Queries for SOC Dashboards

**Version:** 1.0
**Date:** June 2026
**Role:** SOC Analyst / Detection Engineer

---

## 1. Overview
Kibana Query Language (KQL) is the primary method for filtering data in the Elastic Stack. Below are the foundational KQL queries used to power the visualizations described in the Dashboard Design Guide. 

These queries are essential for Threat Hunting and Detection Engineering.

---

## 2. Authentication Queries (Windows Native)

### 2.1 Failed Logins (General)
**Query:**
```kql
event.code: 4625
```
*Use Case:* Base query for the "Total Failed Logins" metric.

### 2.2 Brute Force / Password Spraying Detection
**Query:**
```kql
event.code: 4625 AND winlog.event_data.SubStatus: "0xC000006A"
```
*Use Case:* `0xC000006A` specifically means "Bad Password". If an attacker is guessing passwords, this sub-status will spike.

### 2.3 RDP (Remote Desktop) Logins
**Query:**
```kql
event.code: 4624 AND winlog.event_data.LogonType: 10
```
*Use Case:* Tracks successful remote desktop connections. Analysts should filter this to see if RDP is occurring from external/unexpected IP addresses.

### 2.4 Audit Logs Cleared (Defense Evasion)
**Query:**
```kql
event.code: 1102
```
*Use Case:* High-fidelity indicator of compromise (IOC).

## 3. Process Execution Queries (Sysmon)

### 3.1 Suspicious PowerShell Execution
**Query:**
```kql
event.code: 1 AND process.name: "powershell.exe" AND process.command_line: (*-enc* OR *-EncodedCommand* OR *Bypass* OR *Hidden*)
```
*Use Case:* Detects attackers attempting to bypass execution policies, hide windows, or execute base64 encoded payloads.

### 3.2 Network Connections by System Utilities (LOLBins)
**Query:**
```kql
event.code: 3 AND process.name: ("certutil.exe" OR "bitsadmin.exe" OR "regsvr32.exe" OR "rundll32.exe")
```
*Use Case:* These are native Windows binaries. They have no business making outbound network connections to the internet. If `certutil.exe` connects to an external IP, it is likely downloading malware.

### 3.3 Credential Dumping (LSASS Access)
**Query:**
```kql
event.code: 10 AND winlog.event_data.TargetImage: "*lsass.exe" AND winlog.event_data.GrantedAccess: "0x1010"
```
*Use Case:* Detects tools like Mimikatz requesting specific memory access (`0x1010`) to the Local Security Authority Subsystem Service to dump plaintext passwords or hashes.

## 4. Linux Queries (Filebeat & Auditbeat)

### 4.1 SSH Brute Force (Auth Logs)
**Query:**
```kql
system.auth.ssh.event: "Failed" OR message: "Failed password for"
```
*Use Case:* Identifies failed SSH attempts.

### 4.2 Suspicious Sudo Usage
**Query:**
```kql
message: "NOT in sudoers"
```
*Use Case:* Identifies unprivileged users attempting to run administrative commands.

### 4.3 Interactive Shell Spawning (Auditbeat)
**Query:**
```kql
event.action: "execve" AND process.name: ("bash" OR "sh" OR "dash") AND process.parent.name: ("nginx" OR "apache2" OR "php-fpm")
```
*Use Case:* If a web server process (`nginx`) spawns a bash shell (`bash`), the server has been compromised and a web shell has been executed. This is a critical alert.
