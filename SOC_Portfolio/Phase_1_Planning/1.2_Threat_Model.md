# Baseline Threat Model

**Version:** 1.0
**Date:** June 2026
**Classification:** Internal / Confidential

---

## 1. Introduction
Threat modeling is a core GRC and SOC function used to identify potential threats, vulnerabilities, and the associated risks to the organization's assets. This document outlines the expected adversary behaviors against our simulated enterprise architecture.

## 2. Threat Actors
Based on our asset inventory (Windows endpoints, Linux web servers), we are modeling defenses against the following profiles:

- **Insider Threat (Disgruntled Employee):** Has legitimate access to `WKS-WIN-01`. May attempt privilege escalation, unauthorized data access, or defense evasion (e.g., clearing event logs).
- **External Script Kiddie / Opportunistic Attacker:** Targets internet-facing assets (`SRV-LNX-01`) using automated scanning tools (Nmap) and brute-force attacks (Hydra) against SSH or web portals.
- **Advanced Persistent Threat (APT):** Uses sophisticated techniques (e.g., living-off-the-land binaries, encoded PowerShell, registry persistence) to establish a foothold, escalate privileges, and move laterally across the network.

## 3. Threat Scenarios & MITRE ATT&CK Mapping

The SOC will focus detection engineering efforts around the following high-priority scenarios, mapped to MITRE ATT&CK tactics:

| Scenario | Primary Tactic | Relevant Techniques |
| :--- | :--- | :--- |
| **Brute-Force Authentication** | Credential Access (TA0006) | T1110 (Brute Force), T1110.001 (Password Guessing) |
| **Endpoint Execution** | Execution (TA0002) | T1059.001 (PowerShell), T1047 (WMI) |
| | | |
| **Persistence Mechanisms** | Persistence (TA0003) | T1547.001 (Registry Run Keys), T1053.005 (Scheduled Task) |
| **Defense Evasion** | Defense Evasion (TA0005) | T1070.001 (Clear Windows Event Logs), T1562.001 (Disable Defender) |
| **Lateral Movement** | Lateral Movement (TA0008) | T1569.002 (PsExec), T1021.001 (Remote Desktop Protocol) |

## 4. Attack Surface Analysis
- **`WKS-WIN-01` (Windows 10):** Vulnerable to phishing, malicious payload execution, credential dumping (Mimikatz), and local privilege escalation.
- **`SRV-LNX-01` (Ubuntu):** Vulnerable to web application exploits, SSH brute-forcing, and local privilege escalation (e.g., Sudo abuse).
- **`SRV-SIEM-01` (Elastic):** The crown jewel. If the SIEM is compromised or blinded (logs dropped), the SOC loses visibility. Must be protected by strict firewall rules restricting access to port `9200` only to authorized log shippers.

## 5. Mitigation & Visibility Strategy
To combat these threats, the SOC will deploy:
1. **Sysmon** on Windows to achieve deep visibility into Process Creation (EID 1) and Network Connections (EID 3).
2. **Detection Rules** tailored to the specific TTPs listed above.
3. **Dashboards** to visualize authentication spikes, PowerShell usage, and network flow anomalies.
