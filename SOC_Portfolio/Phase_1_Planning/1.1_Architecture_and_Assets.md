## 1. Enterprise SOC Architecture Overview
The SOC architecture is designed to simulate a modern, centralized log management and analysis environment. It follows a hub-and-spoke model where endpoints generate high-fidelity telemetry that is shipped securely to a central SIEM for indexing and correlation.

### 1.1 Network Topology
The environment operates within an isolated Virtual Local Area Network (VLAN) to safely conduct attack simulations.

- **Subnet:** `192.168.100.0/24` (Example NAT Network)
- **SIEM Zone:** Hosts the Elastic Stack infrastructure.
- **Endpoint Zone:** Hosts the Windows 10 and Ubuntu Linux targets.
- **Attacker Zone:** Hosts the Kali Linux simulation machine.

### 1.2 Technology Stack
All tools selected represent industry standards, allowing for direct skill transfer to enterprise environments.

| Component Category | Technology Selected | Justification |
| :--- | :--- | :--- |
| **SIEM / Data Lake** | Elastic Stack (ELK) | Industry-leading open-source SIEM; highly scalable; KQL is an essential skill. |
| **Log Shippers** | Winlogbeat, Filebeat | Native Elastic agents for lightweight, reliable log transmission. |
| **Endpoint Telemetry** | Sysmon (Windows) | Provides granular process, network, and registry telemetry beyond standard Windows Event Logs. |
| **Target OS (Windows)** | Windows 10 Enterprise | Standard corporate endpoint OS. |
| **Target OS (Linux)** | Ubuntu Server 22.04 LTS | Standard corporate web/application server OS. |
| **Attack Simulation** | Kali Linux, Atomic Red Team | Standard frameworks for mapping simulated attacks to MITRE ATT&CK. |
| **Forensics** | Chainsaw | Rapid event log triage and sigma rule matching. |

## 3. Data Flow Diagram
1. **Generation:** Sysmon and Windows Event Viewer capture system activity on `WKS-WIN-01`.
2. **Collection:** Winlogbeat reads the local EVTX logs.
3. **Transmission:** Winlogbeat securely ships the logs over port `9200` to `SRV-SIEM-01`.
4. **Indexing:** Elasticsearch stores and indexes the JSON documents.
5. **Visualization:** Analysts use Kibana on port `5601` to query KQL, view dashboards, and trigger alerts.
