# Enterprise Detection Rule Library (Part 1/2)

**Version:** 1.0
**Date:** June 2026
**Framework:** MITRE ATT&CK v14

---

## Overview
This library contains enterprise-grade detection rules built for the Elastic SIEM. Each rule includes the KQL logic, MITRE mapping, false positive rates, and investigation guidance, structured identically to tier-1 vendor threat intelligence reports (e.g., CrowdStrike, Microsoft).

---

## Category 1: Credential Access

### Rule 1.01: High Volume of Failed Logins (Password Spraying)
- **Purpose:** Detects an adversary attempting a password spraying attack by attempting a single password against multiple accounts.
- **Severity:** High
- **MITRE Tactic:** Credential Access (TA0006)
- **MITRE Technique:** Brute Force: Password Spraying (T1110.003)
- **Log Source:** Windows Security (Event ID 4625)
- **Elastic Query (KQL):** 
  ```kql
  event.code: 4625 | stats count() by source.ip | where count > 50
  ```
- **Why it exists:** Passwords sprays avoid traditional lockout policies by trying few passwords across many accounts.
- **Expected Behavior:** High volume of 4625 events from a single external IP.
- **False Positives:** Misconfigured application service attempting to authenticate with an old password.
- **Investigation Guide:** Check the `source.ip`. If internal, trace to the host. If external, check firewall logs for successful connections.
- **Containment:** Block the source IP at the edge firewall. Lock compromised accounts.

### Rule 1.02: SSH Brute Force
- **Purpose:** Detects automated SSH guessing attacks against Linux servers.
- **Severity:** Medium
- **MITRE Tactic:** Credential Access (TA0006)
- **MITRE Technique:** Brute Force: Password Guessing (T1110.001)
- **Log Source:** `/var/log/auth.log` (Filebeat)
- **Elastic Query (KQL):** 
  ```kql
  system.auth.ssh.event: "Failed" | stats count() by source.ip | where count > 20
  ```
- **False Positives:** Administrator forgetting their password.
- **Investigation Guide:** Determine if the brute force was followed by an "Accepted password" event from the same IP.

### Rule 1.03: Successful Login After Multiple Failures
- **Purpose:** Identifies a successful brute-force attack.
- **Severity:** Critical
- **MITRE Tactic:** Initial Access (TA0001)
- **MITRE Technique:** Valid Accounts (T1078)
- **Log Source:** Windows Security (4625, 4624)
- **Elastic Query (KQL):** (Requires EQL sequence)
  ```eql
  sequence by user.name with maxspan=5m
    [authentication where event.action == "logon-failed" | repeat > 10]
    [authentication where event.action == "logged-in"]
  ```
- **Investigation Guide:** Treat the account as compromised immediately. Check for subsequent lateral movement (Sysmon EID 3).

### Rule 1.04: LSASS Memory Dumping
- **Purpose:** Detects tools like Mimikatz or ProcDump attempting to extract plaintext credentials from LSASS.
- **Severity:** Critical
- **MITRE Tactic:** Credential Access (TA0006)
- **MITRE Technique:** OS Credential Dumping (T1003.001)
- **Log Source:** Sysmon (Event ID 10 - Process Access)
- **Elastic Query (KQL):** 
  ```kql
  event.code: 10 AND winlog.event_data.TargetImage: "*lsass.exe" AND winlog.event_data.GrantedAccess: ("0x1010" OR "0x1410" OR "0x143a")
  ```
- **False Positives:** EDR/Antivirus software legitimate scanning (Requires whitelisting).
- **Containment:** Isolate the host from the network immediately.

## Category 2: Execution

### Rule 2.01: Suspicious Encoded PowerShell
- **Purpose:** Detects PowerShell executing with base64 encoded commands, a common obfuscation technique.
- **Severity:** High
- **MITRE Tactic:** Execution (TA0002)
- **MITRE Technique:** Command and Scripting Interpreter: PowerShell (T1059.001)
- **Log Source:** Sysmon (Event ID 1), PowerShell (Event ID 4104)
- **Elastic Query (KQL):** 
  ```kql
  event.code: 1 AND process.name: "powershell.exe" AND process.command_line: (*-enc* OR *-e * OR *-EncodedCommand*)
  ```
- **False Positives:** Legitimate deployment scripts (e.g., SCCM, Chocolatey).
- **Investigation Guide:** Decode the base64 string using CyberChef to understand the script's true intent.

### Rule 2.02: PsExec Execution
- **Purpose:** Detects the use of Sysinternals PsExec, heavily utilized for lateral movement and remote execution.
- **Severity:** High
- **MITRE Tactic:** Execution (TA0002)
- **MITRE Technique:** System Services: Service Execution (T1569.002)
- **Log Source:** Sysmon (Event ID 1, 11), Windows System (Event ID 7045)
- **Elastic Query (KQL):** 
  ```kql
  event.code: 7045 AND winlog.event_data.ServiceName: "PSEXESVC"
  ```
- **False Positives:** IT Administrators performing remote management.
- **Investigation Guide:** Verify the source IP and user account. If unauthorized, assume lateral movement has occurred.

### Rule 2.03: Remote PowerShell (WinRM)
- **Purpose:** Detects the execution of PowerShell via Windows Remote Management.
- **Severity:** Medium
- **MITRE Tactic:** Execution (TA0002)
- **MITRE Technique:** Windows Management Instrumentation (T1047)
- **Log Source:** Sysmon (Event ID 1)
- **Elastic Query (KQL):** 
  ```kql
  process.name: "wsmprovhost.exe" AND process.parent.name: "svchost.exe"
  ```
- **Investigation Guide:** Review the child processes spawned by `wsmprovhost.exe`.

### Rule 2.04: Certutil Download
- **Purpose:** Detects `certutil.exe` (a native certificate tool) being abused to download payloads from the internet (LOLBin).
- **Severity:** High
- **MITRE Tactic:** Command and Control (TA0011)
- **MITRE Technique:** Ingress Tool Transfer (T1105)
- **Log Source:** Sysmon (Event ID 1)
- **Elastic Query (KQL):** 
  ```kql
  process.name: "certutil.exe" AND process.command_line: (*-urlcache* OR *-split* OR *-f*)
  ```
- **Investigation Guide:** Extract the URL from the command line and check it against Threat Intelligence (VirusTotal).

### Rule 2.05: WMI Execution
- **Purpose:** Detects execution via Windows Management Instrumentation command-line (wmic).
- **Severity:** Medium
- **MITRE Tactic:** Execution (TA0002)
- **MITRE Technique:** Windows Management Instrumentation (T1047)
- **Log Source:** Sysmon (Event ID 1)
- **Elastic Query (KQL):** 
  ```kql
  process.name: "wmic.exe" AND process.command_line: (*process call create*)
  ```

### Rule 2.06: Interactive Bash from Web Server
- **Purpose:** Detects a web shell execution on a Linux server.
- **Severity:** Critical
- **MITRE Tactic:** Execution (TA0002)
- **MITRE Technique:** Command and Scripting Interpreter (T1059.004)
- **Log Source:** Auditbeat (execve)
- **Elastic Query (KQL):** 
  ```kql
  process.name: ("bash" OR "sh") AND process.parent.name: ("nginx" OR "apache2" OR "www-data")
  ```
- **Investigation Guide:** Inspect web server access logs for file uploads preceding the shell execution. Isolate the server immediately.
