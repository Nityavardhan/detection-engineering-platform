## Category 3: Persistence

### Rule 3.01: Scheduled Task Creation
- **Purpose:** Detects the creation of a Windows scheduled task, often used by malware to survive reboots.
- **Severity:** Medium
- **MITRE Tactic:** Persistence (TA0003)
- **MITRE Technique:** Scheduled Task/Job (T1053.005)
- **Log Source:** Windows Security (Event ID 4698), Sysmon (Event ID 1)
- **Elastic Query (KQL):** 
  ```kql
  event.code: 1 AND process.name: "schtasks.exe" AND process.command_line: "*/create*"
  ```
- **False Positives:** Legitimate software updates (e.g., Google Chrome update task).
- **Investigation Guide:** Review the executable being scheduled. If it points to `AppData`, `Temp`, or `ProgramData`, escalate.

### Rule 3.02: Registry Run Key Modification
- **Purpose:** Detects modification to the Windows Registry "Run" keys for persistence.
- **Severity:** High
- **MITRE Tactic:** Persistence (TA0003)
- **MITRE Technique:** Boot or Logon Autostart Execution (T1547.001)
- **Log Source:** Sysmon (Event ID 12, 13, 14)
- **Elastic Query (KQL):** 
  ```kql
  event.code: (12 OR 13 OR 14) AND winlog.event_data.TargetObject: (*\CurrentVersion\Run* OR *\CurrentVersion\RunOnce*)
  ```

### Rule 3.03: New Local User Creation
- **Purpose:** Detects the creation of a new local user account.
- **Severity:** Medium
- **MITRE Tactic:** Persistence (TA0003)
- **MITRE Technique:** Create Account: Local Account (T1136.001)
- **Log Source:** Windows Security (Event ID 4720)
- **Elastic Query (KQL):** 
  ```kql
  event.code: 4720
  ```
- **Investigation Guide:** Verify if the account creation aligns with an approved IT change request.

### Rule 3.04: Admin Group Membership Changes
- **Purpose:** Detects a user being added to a highly privileged local group.
- **Severity:** High
- **MITRE Tactic:** Privilege Escalation (TA0004)
- **MITRE Technique:** Valid Accounts (T1078)
- **Log Source:** Windows Security (Event ID 4732)
- **Elastic Query (KQL):** 
  ```kql
  event.code: 4732 AND winlog.event_data.TargetUserName: ("Administrators" OR "Remote Desktop Users")
  ```

### Rule 3.05: Startup Folder Persistence
- **Purpose:** Detects a file being dropped into the Windows Startup folder.
- **Severity:** High
- **MITRE Tactic:** Persistence (TA0003)
- **MITRE Technique:** Boot or Logon Autostart Execution (T1547.001)
- **Log Source:** Sysmon (Event ID 11 - File Create)
- **Elastic Query (KQL):** 
  ```kql
  event.code: 11 AND winlog.event_data.TargetFilename: "*\\Start Menu\\Programs\\Startup\\*"
  ```

## Category 4: Defense Evasion

### Rule 4.01: Event Log Clearing
- **Purpose:** Detects the deliberate clearing of Windows Event Logs to hide malicious activity.
- **Severity:** Critical
- **MITRE Tactic:** Defense Evasion (TA0005)
- **MITRE Technique:** Indicator Removal on Host: Clear Windows Event Logs (T1070.001)
- **Log Source:** Windows Security (Event ID 1102), Windows System (Event ID 104)
- **Elastic Query (KQL):** 
  ```kql
  event.code: (1102 OR 104)
  ```
- **False Positives:** None. IT should not clear security logs manually in a governed environment.

### Rule 4.02: Windows Defender Disabled
- **Purpose:** Detects PowerShell commands or Registry edits intended to disable Real-Time Monitoring.
- **Severity:** High
- **MITRE Tactic:** Defense Evasion (TA0005)
- **MITRE Technique:** Impair Defenses: Disable or Modify Tools (T1562.001)
- **Log Source:** Sysmon (Event ID 1)
- **Elastic Query (KQL):** 
  ```kql
  process.name: "powershell.exe" AND process.command_line: "*Set-MpPreference -DisableRealtimeMonitoring $true*"
  ```

### Rule 4.03: Firewall Disabled
- **Purpose:** Detects the execution of netsh to disable the host firewall.
- **Severity:** High
- **MITRE Tactic:** Defense Evasion (TA0005)
- **MITRE Technique:** Impair Defenses: Disable or Modify System Firewall (T1562.004)
- **Log Source:** Sysmon (Event ID 1)
- **Elastic Query (KQL):** 
  ```kql
  process.name: "netsh.exe" AND process.command_line: "*advfirewall set allprofiles state off*"
  ```

## Category 5: Discovery & Lateral Movement

### Rule 5.01: Network Scanning (Reconnaissance)
- **Purpose:** Detects rapid, sequential outbound connections to multiple IP addresses, indicating port scanning.
- **Severity:** Medium
- **MITRE Tactic:** Discovery (TA0007)
- **MITRE Technique:** Network Service Discovery (T1046)
- **Log Source:** Sysmon (Event ID 3)
- **Elastic Query (KQL):** (Requires correlation or thresholding)
  ```kql
  event.code: 3 | stats distinct_count(destination.ip) as dest_ips by source.ip | where dest_ips > 100
  ```

### Rule 5.02: Suspicious RDP Activity (Lateral Movement)
- **Purpose:** Detects RDP connections originating from non-standard IT administration subnets.
- **Severity:** High
- **MITRE Tactic:** Lateral Movement (TA0008)
- **MITRE Technique:** Remote Services: Remote Desktop Protocol (T1021.001)
- **Log Source:** Windows Security (Event ID 4624)
- **Elastic Query (KQL):** 
  ```kql
  event.code: 4624 AND winlog.event_data.LogonType: 10 AND NOT source.ip: ("10.0.50.*" OR "192.168.1.*")
  ```
- **Investigation Guide:** Check the user logging in. If it is a service account, immediately isolate the host (service accounts should never use interactive RDP).
