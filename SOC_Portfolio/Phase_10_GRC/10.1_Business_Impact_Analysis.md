**Version:** 1.0

**Role:** GRC Analyst / Business Continuity Manager
**Framework:** ISO 22301 (Business Continuity Management)

---

## 1. What is a Business Impact Analysis?

A BIA identifies the organization's critical business functions and determines the impact if those functions are disrupted. It answers two key questions:
- **How long can we survive without this system?** (Recovery Time Objective — RTO)
- **How much data can we afford to lose?** (Recovery Point Objective — RPO)

## 3. Impact Assessment

### 3.1 Recovery Objectives

| Function | RTO (Max Downtime) | RPO (Max Data Loss) | Financial Impact (per hour) | Reputational Impact |
| :--- | :--- | :--- | :--- | :--- |
| BF-001 Customer Transactions | **1 hour** | **0 minutes** (real-time replication) | $50,000/hr | **Severe** — customer churn |
| BF-002 Employee Authentication | **2 hours** | **N/A** | $20,000/hr (lost productivity) | Moderate |
| BF-003 Email Communication | **4 hours** | **1 hour** | $5,000/hr | Low–Moderate |
| BF-004 Financial Reporting | **24 hours** | **4 hours** | $10,000/hr (end of quarter) | Low |
| BF-005 Security Monitoring | **30 minutes** | **0 minutes** | Unquantifiable (blind spot) | **Severe** — regulatory |
| BF-006 Internal File Sharing | **8 hours** | **24 hours** | $2,000/hr | Negligible |
| BF-007 Software Development | **24 hours** | **4 hours** | $3,000/hr | Low |

### 3.2 Key Findings
- **BF-001 (Customer Transactions)** and **BF-005 (Security Monitoring)** are the two most critical functions with the lowest acceptable downtime.
- If the SIEM (BF-005) goes offline for more than 30 minutes during an active incident, the SOC is effectively blind. This justifies future investment in SIEM high-availability.
- **BF-002 (Authentication)** has a cascading impact—if Active Directory is down, employees cannot log into any system, crippling all other business functions.

## 4. Backup & Recovery Strategy

| Function | Backup Method | Backup Frequency | Backup Location | Tested? |
| :--- | :--- | :--- | :--- | :--- |
| BF-001 Customer Transactions | Database replication + daily snapshot | Real-time + Daily | Off-site cloud (encrypted) | Yes (quarterly) |
| BF-002 Employee Authentication | AD System State backup | Daily | Off-site NAS (air-gapped) | Yes (monthly) |
| BF-003 Email Communication | O365 native retention + Veeam | Continuous | Microsoft Cloud | Yes (annually) |
| BF-004 Financial Reporting | File-level backup | Daily | Off-site NAS | No — **Gap identified** |
| BF-005 Security Monitoring | Elasticsearch snapshots | Every 6 hours | Local + off-site | Yes (monthly) |

---

## 5. Recommendations

1. **Implement SIEM High-Availability:** BF-005 has a 30-minute RTO. A single-node Elasticsearch cluster is a single point of failure. Recommend deploying a multi-node cluster.
2. **Test Financial Reporting Backups:** BF-004 backups have never been tested. Schedule a quarterly restore test.
3. **Document Disaster Recovery Procedures:** Create step-by-step runbooks for restoring each critical function from backup.
