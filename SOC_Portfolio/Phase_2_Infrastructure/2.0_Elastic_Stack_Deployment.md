# Standard Operating Procedure: Elastic Stack (SIEM) Deployment

**Version:** 1.0
**Date:** June 2026
**Role:** SOC Engineering / Infrastructure

---

## 1. Objective
This document outlines the enterprise-standard procedure for deploying the Elastic Stack (Elasticsearch and Kibana) as the centralized Security Information and Event Management (SIEM) solution for the SOC.

## 2. Prerequisites
- **OS:** Ubuntu Server 22.04 LTS (`SRV-SIEM-01`)
- **Resources:** Minimum 8GB RAM, 4 CPU Cores, 60GB Disk
- **Network:** Static IP assigned (e.g., `192.168.100.10`)
- **Privileges:** `root` or `sudo` access.

---

## 3. Deployment Steps

### 3.1. System Preparation
Before installing Elastic, the host system must be updated and necessary dependencies installed.

```bash
# Update package repositories
sudo apt-get update && sudo apt-get upgrade -y

# Install prerequisites
sudo apt-get install apt-transport-https uuid-runtime default-jre -y
```

> **Why this matters:** Elasticsearch runs on the Java Virtual Machine (JVM). Installing `default-jre` ensures the runtime environment is present.

### 3.2. Add Elastic Repository
We use the official Elastic APT repository to ensure we get the latest stable version and automatic updates.

```bash
# Download and install the public signing key
wget -qO - https://artifacts.elastic.co/GPG-KEY-elasticsearch | sudo gpg --dearmor -o /usr/share/keyrings/elasticsearch-keyring.gpg

# Save the repository definition
echo "deb [signed-by=/usr/share/keyrings/elasticsearch-keyring.gpg] https://artifacts.elastic.co/packages/8.x/apt stable main" | sudo tee /etc/apt/sources.list.d/elastic-8.x.list

# Update package index
sudo apt-get update
```

### 3.3. Install and Configure Elasticsearch
Elasticsearch is the data lake where all our logs will be indexed and stored.

```bash
sudo apt-get install elasticsearch -y
```

**Configuration ( `/etc/elasticsearch/elasticsearch.yml` ):**
Modify the configuration file to allow network binding so other machines can send logs to it.

```yaml
# Set the network host to the server's IP (or 0.0.0.0 for all interfaces)
network.host: 0.0.0.0

# Disable security features for local lab environments ONLY
# In a true production environment, xpack.security must be enabled!
xpack.security.enabled: false
```

**Start the Service:**
```bash
sudo systemctl enable elasticsearch
sudo systemctl start elasticsearch
```

**Verification:**
Run `curl -X GET "localhost:9200"` to verify Elasticsearch is responding.

### 3.4. Install and Configure Kibana
Kibana is the frontend visualization layer where analysts write KQL queries and view dashboards.

```bash
sudo apt-get install kibana -y
```

**Configuration ( `/etc/kibana/kibana.yml` ):**
```yaml
# Allow Kibana to be accessed from any IP on the network
server.host: "0.0.0.0"

# Point Kibana to the local Elasticsearch instance
elasticsearch.hosts: ["http://localhost:9200"]
```

**Start the Service:**
```bash
sudo systemctl enable kibana
sudo systemctl start kibana
```

---

## 4. Verification and Validation

1. **Access the Dashboard:** Open a web browser on your host machine and navigate to `http://192.168.100.10:5601`.
2. **Troubleshooting:** 
   - If the dashboard does not load, verify that the Ubuntu firewall (UFW) allows traffic on port 5601: `sudo ufw allow 5601/tcp`.
   - Check Kibana logs if Elasticsearch connection fails: `sudo journalctl -u kibana -f`.

## 5. Security Notice
> **[WARNING]** 
> The configuration above disables `xpack.security.enabled` to simplify the initial lab setup. In an enterprise environment, traffic between log shippers and Elasticsearch must be encrypted using TLS, and API keys must be used for authentication.
