# Standard Operating Procedure: Windows Endpoint Sensor Deployment

**Version:** 1.0
**Date:** June 2026
**Role:** Detection Engineering / Endpoint Operations

---

## 1. Objective
This SOP dictates the installation and configuration of endpoint telemetry agents (Sysmon and Winlogbeat) on Windows systems to feed high-fidelity logs into the SIEM.

## 2. Prerequisites
- **OS:** Windows 10 Enterprise (`WKS-WIN-01`)
- **Privileges:** Local Administrator access.
- **Connectivity:** Must be able to reach `SRV-SIEM-01` on port `9200`.

---

## 3. Sysmon Deployment
Standard Windows Event Logs do not capture deep process-level activity (e.g., command-line arguments, network connections by process). We deploy Microsoft Sysinternals **Sysmon** to bridge this gap.

### 3.1. Download and Installation
1. Download Sysmon from Microsoft Docs.
2. Download the enterprise-grade SwiftOnSecurity Sysmon configuration file. This file filters out noisy operating system events while capturing critical indicators of compromise.

**Execution (Run in elevated PowerShell):**
```powershell
# Install Sysmon with the configuration file, accepting the EULA automatically
.\Sysmon64.exe -accepteula -i sysmonconfig-export.xml
```

> **Why this matters:** A default Sysmon installation logs *everything*, which will overwhelm your SIEM and cost thousands of dollars in licensing. A tuned XML config is the hallmark of a mature Detection Engineering program.

---

## 4. Winlogbeat Deployment
Winlogbeat is the elastic agent responsible for reading EVTX files and forwarding them to Elasticsearch.

### 4.1. Download and Installation
1. Download Winlogbeat (ZIP format) from the official Elastic website.
2. Extract the contents to `C:\Program Files\Winlogbeat`.

### 4.2. Configuration (`winlogbeat.yml`)
Modify the `winlogbeat.yml` file to capture Security, System, and Sysmon logs, and point the output to our SIEM.

```yaml
winlogbeat.event_logs:
  - name: Application
    ignore_older: 72h
  - name: System
  - name: Security
  - name: Microsoft-Windows-Sysmon/Operational

output.elasticsearch:
  # Replace with the IP of your SRV-SIEM-01
  hosts: ["192.168.100.10:9200"]

setup.kibana:
  host: "192.168.100.10:5601"
```

### 4.3. Service Installation and Verification
Open an administrative PowerShell prompt and execute:

```powershell
cd 'C:\Program Files\Winlogbeat'

# Install the Windows service
.\install-service-winlogbeat.ps1

# Test the configuration file for syntax errors
.\winlogbeat.exe test config -c .\winlogbeat.yml -e

# Test the connection to the Elasticsearch server
.\winlogbeat.exe test output

# Start the service
Start-Service winlogbeat
```

---

## 5. Validation in Kibana
1. Navigate to Kibana (`http://192.168.100.10:5601`).
2. Go to **Management > Stack Management > Index Management**. You should see a new index created, starting with `winlogbeat-`.
3. Go to **Discover** and create a Data View for `winlogbeat-*`. You should now see live events streaming from your Windows endpoint.
