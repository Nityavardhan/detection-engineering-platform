# Standard Operating Procedure: Linux Endpoint Sensor Deployment

**Version:** 1.0
**Date:** June 2026
**Role:** Detection Engineering / Endpoint Operations

---

## 1. Objective
This SOP dictates the installation and configuration of endpoint telemetry agents (Filebeat and Auditbeat) on Linux systems (e.g., web servers, databases) to feed system, authentication, and execution logs into the SIEM.

## 2. Prerequisites
- **OS:** Ubuntu Server 22.04 LTS (`SRV-LNX-01`)
- **Privileges:** `sudo` access.
- **Connectivity:** Must be able to reach `SRV-SIEM-01` on port `9200`.

---

## 3. Filebeat Deployment
Filebeat is a lightweight shipper that forwards and centralizes log data. We will use it to capture `/var/log/auth.log` (SSH logins) and `/var/log/syslog`.

### 3.1. Installation
Assuming the Elastic repository is already added (see Elastic Stack deployment guide):

```bash
sudo apt-get update
sudo apt-get install filebeat -y
```

### 3.2. Configuration (`/etc/filebeat/filebeat.yml`)
Modify the configuration to point to the SIEM.

```yaml
filebeat.inputs:
- type: filestream
  id: linux-system-logs
  enabled: true
  paths:
    - /var/log/auth.log
    - /var/log/syslog

output.elasticsearch:
  # Replace with the IP of your SRV-SIEM-01
  hosts: ["192.168.100.10:9200"]

setup.kibana:
  host: "192.168.100.10:5601"
```

### 3.3. Enable Modules
Filebeat includes built-in modules that parse and format logs automatically.

```bash
# Enable the system module (auth, syslog, etc.)
sudo filebeat modules enable system

# Setup the index template and dashboards (Run this once)
sudo filebeat setup -e

# Start the service
sudo systemctl enable filebeat
sudo systemctl start filebeat
```

---

## 4. Auditbeat Deployment
While Filebeat captures text logs, Auditbeat integrates with the Linux Audit Framework (`auditd`) to capture process execution, file integrity monitoring, and network socket creation. This is the Linux equivalent of Windows Sysmon.

### 4.1. Installation
```bash
sudo apt-get install auditbeat -y
```

### 4.2. Configuration (`/etc/auditbeat/auditbeat.yml`)
Configure Auditbeat to connect to Elasticsearch.

```yaml
auditbeat.modules:
- module: auditd
  audit_rules: |
    # Log 64-bit processes executing
    -a always,exit -F arch=b64 -S execve -k exec

- module: file_integrity
  paths:
  - /bin
  - /usr/bin
  - /sbin
  - /usr/sbin
  - /etc

- module: system
  datasets:
    - process
    - network
    - socket

output.elasticsearch:
  hosts: ["192.168.100.10:9200"]
```

### 4.3. Start the Service
```bash
# Set up dashboards and indexes
sudo auditbeat setup -e

# Start the service
sudo systemctl enable auditbeat
sudo systemctl start auditbeat
```

---

## 5. Validation in Kibana
1. Navigate to Kibana (`http://192.168.100.10:5601`).
2. Go to **Management > Stack Management > Index Management**. You should see new indices created, starting with `filebeat-` and `auditbeat-`.
3. Try SSHing into the Linux machine and observe the events populating in the `filebeat-*` Data View under the `system.auth` module.
