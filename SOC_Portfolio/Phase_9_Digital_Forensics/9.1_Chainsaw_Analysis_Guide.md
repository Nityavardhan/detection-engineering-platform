# Chainsaw: Rapid Windows Event Log Analysis

**Version:** 1.0
**Date:** June 2026
**Role:** Digital Forensics Analyst / Threat Hunter

---

## 1. What is Chainsaw?

Chainsaw is a free, open-source tool developed by WithSecure (formerly F-Secure) for rapidly searching and analyzing Windows Event Log files (`.evtx`). It is purpose-built for incident response triage and threat hunting.

**Why Chainsaw over manual EVTX review?**
- A single Windows machine generates **thousands** of events per hour.
- Manually opening each `.evtx` file in Event Viewer is impractical during a time-sensitive incident.
- Chainsaw uses **Sigma rules** to automatically scan for known attack patterns, returning only high-value hits.

---

## 2. Installation

Chainsaw is a standalone Rust binary—no installation required.

```bash
# Download the latest release from GitHub
# https://github.com/WithSecureLabs/chainsaw/releases

# On Linux/macOS:
chmod +x chainsaw
./chainsaw --version

# On Windows:
.\chainsaw.exe --version
```

## 3. Core Commands

### 3.1 Hunt Mode (Sigma Rule Scanning)
This is the primary forensics use case. Chainsaw scans EVTX files against a library of Sigma rules and outputs only the matches.

```bash
# Hunt through all EVTX files in a directory using bundled Sigma rules
./chainsaw hunt /path/to/evtx/files/ --sigma /path/to/sigma/rules/ --mapping mappings/sigma-event-logs-all.yml

# Example with our project's EVTX samples:
.\chainsaw.exe hunt C:\Cases\Evidence\evtx\ --sigma .\sigma\rules\ --mapping .\mappings\sigma-event-logs-all.yml --output results.json --json
```

**What this does:**
1. Reads every `.evtx` file in the specified directory.
2. Applies every Sigma rule in the specified rules directory.
3. Outputs a list of matches with timestamps, Event IDs, and rule metadata.

### 3.2 Search Mode (Keyword Hunting)
Search for a specific string across all EVTX files.

```bash
# Search for a specific username across all logs
./chainsaw search "suspicious_user" /path/to/evtx/files/

# Search for a specific IP address
./chainsaw search "192.168.100.99" /path/to/evtx/files/

# Search for a specific process name
./chainsaw search "mimikatz" /path/to/evtx/files/
```

### 3.3 Dump Mode (Full Event Extraction)
Extract all events of a specific type for detailed analysis.

```bash
# Dump all Logon events (Event ID 4624)
./chainsaw dump /path/to/evtx/files/ --event-id 4624

# Dump all Process Creation events (Sysmon Event ID 1)
./chainsaw dump /path/to/evtx/files/ --event-id 1
```

## 4. Interpreting Chainsaw Output

Chainsaw outputs results in a structured format. Here is an example of a Sigma rule match:

```
[+] Group: Sigma
┌──────────────────────┬──────────────────────────────────────────────┐
│ Timestamp            │ 2026-06-15T14:32:01.000Z                     │
│ Detection Rule       │ Suspicious Encoded PowerShell Command         │
│ Level                │ high                                          │
│ Event ID             │ 1                                             │
│ Computer             │ WKS-WIN-01                                    │
│ User                 │ CORP\jsmith                                   │
│ Command Line         │ powershell.exe -enc ZWNobyAiaGVsbG8i          │
│ Parent Process       │ cmd.exe                                       │
└──────────────────────┴──────────────────────────────────────────────┘
```

**How to read this:**
- **Timestamp:** When the malicious activity occurred.
- **Detection Rule:** The Sigma rule that matched (tells you the *type* of attack).
- **Level:** Severity (informational, low, medium, high, critical).
- **Command Line:** The exact command the attacker ran.
- **Parent Process:** What spawned the malicious process (crucial for determining the kill chain).

## 5. Building a Forensic Timeline

After running Chainsaw, combine the results into a chronological **Attack Timeline**:

| Timestamp | Event | Source | Details |
| :--- | :--- | :--- | :--- |
| 14:28:00 | SSH Brute Force begins | `auth.log` | 50+ failed SSH attempts from `192.168.100.99` |
| 14:31:15 | SSH Login Successful | `auth.log` | `root` login from `192.168.100.99` |
| 14:32:01 | Encoded PowerShell executed | Sysmon EID 1 | `powershell -enc ZWNobyAi...` on `WKS-WIN-01` |
| 14:33:45 | LSASS memory accessed | Sysmon EID 10 | `mimikatz.exe` opened `lsass.exe` |
| 14:35:00 | New user created | Security EID 4720 | Account `backdoor_admin` created |
| 14:36:30 | User added to Admins | Security EID 4732 | `backdoor_admin` added to `Administrators` |
| 14:38:00 | Security log cleared | Security EID 1102 | Attacker attempted to cover tracks |

This timeline is the core deliverable of any forensic investigation. It tells the complete story of the attack from initial access to persistence.
