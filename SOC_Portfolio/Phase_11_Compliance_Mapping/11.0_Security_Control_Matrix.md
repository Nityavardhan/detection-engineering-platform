# Security Control Matrix: Detection-to-Compliance Mapping

**Version:** 1.0
**Date:** June 2026
**Role:** GRC Analyst / Compliance Officer

---

## 1. Overview

A Security Control Matrix is the bridge between technical security operations and regulatory compliance. It demonstrates, in a single document, that every detection rule, policy, and control we have deployed satisfies specific requirements from industry frameworks.

This matrix maps our SOC's detection rules, policies, and operational procedures to three industry-standard frameworks:
- **MITRE ATT&CK** — Threat-based mapping (what adversary behavior does this detect?)
- **NIST Cybersecurity Framework (CSF) 2.0** — Risk-based mapping (what security function does this serve?)
- **ISO 27001:2022** — International information security standard (what Annex A control does this satisfy?)
- **CIS Controls v8** — Actionable best practices (what specific safeguard does this implement?)

## 4. NIST CSF Coverage Summary

The NIST Cybersecurity Framework organizes security functions into five pillars. Our SOC coverage across these pillars:

| NIST CSF Function | Our Coverage | Key Evidence |
| :--- | :--- | :--- |
| **Govern (GV)** | ✅ Covered | Project Charter, Security Policies, Risk Register |
| **Identify (ID)** | ✅ Covered | Asset Inventory, Threat Model, Risk Assessment, BIA |
| **Protect (PR)** | ✅ Covered | Access Control Policy, Password Policy, Sysmon/Winlogbeat configuration |
| **Detect (DE)** | ✅ Covered | 20 Detection Rules, SIEM Dashboards, Threat Hunting Playbooks |
| **Respond (RS)** | ✅ Covered | 15 IR Playbooks (NIST 800-61), Digital Forensics Methodology |
| **Recover (RC)** | ✅ Covered | Backup Policy, BIA with RTO/RPO, Lessons Learned in IR playbooks |
